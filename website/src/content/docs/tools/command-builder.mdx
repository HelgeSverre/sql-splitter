## Tips

- **Start simple**: Begin with required options, then add flags as needed
- **Use aliases**: Commands have short aliases (e.g., `sp` for `split`) for faster typing
- **Glob patterns**: Quote patterns like `"*.sql"` to prevent shell expansion
- **Compression**: Input files can be `.gz`, `.bz2`, `.xz`, or `.zst` - no extra flags needed

## Common Patterns

### Development Data Pipeline

```bash
sql-splitter sample prod.sql.gz --percent 10 --preserve-relations | \
  sql-splitter redact - --hash "*.email" --fake "*.name" -o dev.sql
```

### CI/CD Validation

```bash
sql-splitter validate "migrations/*.sql" --strict --json --fail-fast
```

### Database Migration

```bash
sql-splitter convert mysql.sql --to postgres | psql "$DATABASE_URL"
```
